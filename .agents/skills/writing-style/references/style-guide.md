# 文章スタイル

普段の Markdown 文章スタイルでテキストを生成 または推敲する。
日常的なメモ・ドキュメント向けのスタイル。

## 一人称, 語調
- 一人称は「僕」
- 敬語は使わない。断定口調でも馴れ馴れしくもない フラットなトーン
- 読者に語りかける感覚はあるが 講義調にはしない
- 人名の敬称は「〇〇 さん」を使う。「〇〇氏」は使わない
- 列挙の区切りは `・` を使わず `, ` を使う
- 全角ハイフン「—」「―」は使わない。半角の `-` か `:` で代替する
- かっこは半角 `()` を使う。全角 `（）` は使わない
- `【】`, `「」`, `[]` は文章中では基本使わない
  - 会話例や引用を示すときは半角ダブルクォート `"..."` を使う
  - 概念語をくくる必要があるときは `__太字__` か くくらずそのまま書く
- 書籍タイトルには `『』` を付ける
- 文末の句点 `。` は基本使わない。体言止めや動詞止めで自然に終わらせる
- 読点 `、` はあまり使わない。文の区切りや間は半角スペースで表現する
  - `〜だが、` `〜のもと、` のような接続の間も スペースに置き換える
- `→` は使わない。`->` にするか 日本語で流れを表現する
  - 例: `取得 -> 比較` / `取得して比較する` / `取得して、比較する`
- `〜することができる` は使わず `〜できる` にする
- `〜し、` は使わず `〜して` にする
- 迷ったときは「一人称が僕で スペースで区切られた読みやすい文章」を意識する

## 文章のリズム
- 1文が長くなりそうなら 途中で改行して区切る
- 文節の間にスペースを入れて読みやすくする
  - 例: 「コードは __必ず変更される__ という前提で 設計する」
- 1つの段落に詰め込まず 1アイデア = 1〜2文 を目安にする
- 文章が続いているのに Markdown の強制改行 (`  ` + 改行) で途中で折らない
  - 1文が長くなるなら 別の文に分けて短くする
  - 別のアイデアなら 空行で段落を分ける

## 強調
- 重要なキーワードや概念は `__太字__` (アンダースコア2つ) で囲む
- `**太字**` は使わない
- 強調は多用しない。本当に伝えたい1点だけに絞る

## 説明のパターン
難しい概念はまずざっくり定義してから 詳細に入る。

```markdown
### 〇〇とは
ざっくり言うなら __〇〇のこと__ です。

正確には 〇〇であり 〇〇という特性を持つ。
```

「ざっくり言うなら」のフレーズは積極的に使ってよい。

## 箇条書き
- 数字で表せるものは "フェーズ 1" "レベル 2" のような冠語を付けず "1." "2." とシンプルに書く
- 体言止め か 短い動詞止め で統一する
- 長い箇条書きを並べるくらいなら 文章で書く
- ネストは最大2階層まで

## 見出し構成
大げさな構成は作らない。フラットで実用的なものを好む。

```markdown
# タイトル

## 背景 or なぜやるか

## 本題

## まとめ or 結論
```

目次は不要。CSS も不要。
セクションが多いときだけ シンプルな箇条書き目次を付ける程度。

### `---` の使いどころ
- 見出し (`##` など) の直前には置かない。見出しが区切りを兼ねるので不要
- セクション内の内容が大きくなってきたときに 章の内部区切りとして適宜入れる

### 見出しと改行
- 見出しの直後には空行を入れず 内容をすぐ続ける
- 見出しの前の行に文章があるなら 空行を入れてもよい

## コード例
- 悪い例と良い例のペアで示すことを好む
- 言語指定付きのコードブロックを使う
- コメントは最小限にして コードで語る

## テーブル
比較や指標の表は Markdown テーブルで書く。ヘッダーは短くする。

```markdown
| 指標 | 意味 |
| - | - |
| 〇〇 | 〇〇 |
```

## 参考文献セクション
外部リンクは必ず `<a>` タグでセキュアに書く。書籍は著者名を添える。
リンクを1行ずつ表示するときだけ 行末に半角スペース2個を入れる。

```markdown
## その他 参考文献
<a href="URL" target="_blank" rel="noopener noreferrer">書籍タイトル</a> (著: 著者名)  
<a href="URL" target="_blank" rel="noopener noreferrer">記事タイトル</a>  
```
